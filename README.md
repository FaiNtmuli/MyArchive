**前言**

按照道理来讲，AI时代下的学习应当可以更加效率，但是我们总是容易陷于这种浮躁。所以我应当在其中找到一个合适的平衡点。当前我的一个常见困境是：

1.总喜欢翘课并觉得自学更好，但是自学又喜欢摆烂，很浮躁不愿意练习 
2.又觉得AI确实应该合适运用，很多内容我是不需要真的学习的，譬如正则表达式的具体规则，我只需要交给AI并表述清楚需求。 
3.自己 网课/AI学习 的时候过一到两周（个人时常断电，不知道为啥动力是断断续续的，实际上我现在缺乏做任何事的信心，但是还是先不苛责自己了，因为一直都在痛苦地摆烂，再苛责只会摆更狠）就会全忘光

那么问题就在于，哪些是我可以交给AI的，哪些不能。如果不能，我应该准备哪些练习；如果能，我应该如何表达需求设计prompt。————以及，为了避免自己的一两周后没学就废掉，我需要给我自己留下一个极简记录（教程）。

**这个记录应当如何设计**

因为必须侧重于实践侧，一个合适的方法应当是通过一个个project的保留，但是与此同时基础知识如果不会的话，后面看到这些内容我还是会蒙（尽管现在有AI会好很多），所以合适的方法应该是通过某个项目来学习一些基础知识，再通过核心练习来帮助一两周后废掉的我自己完成重建。

故设计为：

Project - Model/Practice/Answer/README

Project 以基础知识命名，表示学习某个基础知识所需要的内容。

Model 表示模板，其中部分包含 可AI 标记（以及对AI的表达prompt）。譬如正则表达式相对应的式子。但在模板后端部分仍会给出一串内容，提供练习（提取某部分等）与答案，实践中可以直接AI。

Practice 为除模板外的 自己尝试/教程中非例题中的练习 。通常跟模板相仿。

Answer 为Practice答案。

README 内记载一些其他特殊事项（如：学习来源/日期记录/碎碎念 等）

**关于AI利用方法**

我们只需要将注明 可AI 的部分统统丢进AI让其生成。不需要真的对于可AI部分掌握非常熟练。如果AI无法成功生成，我们只需要将Model里面的练习提供给AI让他重新临时学习一下即可。又如python里的 strip/split 等函数，这些同样是可以不用记的（虽然并不难），但是我们仍然需要搞清楚我们可以如何表达清楚我们的需求（等问AI问多了我应该自然而然彻底掌握这几个函数了）

**关于Model**

尽量写的“可读性”高一些，这里是指，做某个事情可以分成哪几个步骤，某个容易忘记含义的东西（譬如bs4提取的中间一小段部分），如何用简要的语言“概括”，而不仅是代码名。

**关于学习方式**

1.快速地看完某个 网课 / 教程 。
目的是获得整体认知，先不管掌握熟练度。这一步要点在快，主要记录需要完成的练习（回看的章节），以及简要思考后续结构。

2.完成相应的练习以及所需要的“极简记录”。
目的是提高熟练度和自己的真正意义上的实践能力，同时也有利于自己通过费曼学习法进行学习。

**其他内容**
